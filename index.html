Timer URL Documentation & User Guide

This application reads configuration rules directly from your browser's address bar. You can mix and match parameters using URL queries (?) and ampersands (&).

1. Relative Countdown Timers (Shorthand)

Enter a number followed by time units to start a quick countdown from the moment the page loads.

  • Supported Units: h (hours), m (minutes), s (seconds).
  • Format Example: index.html?1h30m15s sets a timer for 1 hour, 30 minutes, and 15 seconds.
  • Pure Minute Fallback: If you pass a single raw number without units (e.g., index.html?45), the application automatically treats it as minutes (45 minutes).

2. Absolute Target Timers (Specific Time / Date)

You can target specific milestones. The application automatically calculates the remaining duration.

  • Time Only (24-Hour Clock): index.html?15:30 targets 3:30 PM. If that time has already passed today, it automatically schedules the countdown for 3:30 PM tomorrow.
  • Date Only (Midnight Target): index.html?2026-12-25 targets 12:00 AM on Christmas Day, 2026.
  • Exact Date & Time Combo: index.html?2026-05-07&14:00 targets exactly 2:00 PM on May 7th, 2026.

3. Adding Labels

Give your countdown context by displaying a custom header above the clock ticking down.

  • Parameter: label=Your+Text
  • Example: index.html?20m&label=Pizza+Time displays "PIZZA TIME" in clean, bold lettering above a 20-minute timer.
  • Select punctuation for labels is not supported.

4. Display Customization (Font Toggling)

Control the typeface layout of this information screen using the font modifier parameter.

  • font=1: Renders the documentation text in the retro DSEG14 Modern Italic (Digital) font.
  • font=2: Renders the documentation text in the clean, modern Helvetica Neue Medium font.

5. Color Customization (Text)

Change the glowing text color of your timer by passing a standard 6-character hex code or HTML color name.

  • Parameter: color=HEX_OR_NAME
  • Examples: index.html?10m&color=ff0055 (Neon Pink) or index.html?5m&color=cyan

6. Background Color Customization

Change the background color of the timer page by passing a standard 6-character hex code, 3-character hex code, or HTML color name.

  • Parameter: bg=HEX_OR_NAME
  • Examples: index.html?10m&bg=112233 (Dark blue) or index.html?5m&bg=black
  • Strict Validation: If you pass an invalid color string (e.g., bg=banana or bg=#GGGGGG), the timer will explicitly fail with a red error message telling you exactly what went wrong. The app will not silently fallback to the default black.

7. Error Diagnostics Console

If any validation errors are detected, the timer will display a comprehensive error console above the timer face.

  • Multiple Errors: All errors are collected and displayed as a numbered list.
  • Error Header: Shows 1 ERROR WAS FOUND or X ERRORS WERE FOUND.
  • Color Coding: Errors are displayed in bright red. The timer text and label remain in your chosen color (default green).
  • Auto-Shrink: If the error list is long, the text automatically shrinks to fit the screen, pushing the timer down.
  • Calendar Accuracy: Dates are validated against the actual Gregorian calendar. 2026-04-31 will explicitly fail with a human-readable error.

8. Millisecond Precision Display

Add live millisecond precision to your countdown by passing the ms-digit parameter. This will append a decimal point and the specified number of digits to your timer.

  • Parameter: ms-digit=INTEGER (Acceptable values: 0 through 4)
  • Example: index.html?10m&ms-digit=3 displays 09:59:59.123 (with 3 digits of milliseconds).
  • Strict Validation: If you provide a number outside the 0–4 range (e.g., ms-digit=5 or ms-digit=-1), the timer will explicitly fail with a red error message telling you exactly what went wrong. The app will not silently fallback to 0.
  • ms-digit=4 Warning: Setting ms-digit=4 forces the timer to update 1,000 times per second. The app will pause and show a confirmation modal warning about high CPU usage and battery drain before proceeding.

9. Single-Unit Display Mode

Instead of the traditional HH:MM:SS format, you can force the timer to count down exclusively in one specific unit (e.g., just seconds, or just years).

  • Parameter: unit=STRING
  • Supported Units: ms, seconds, minutes, hours, days, weeks, months, years
  • Example: index.html?1h&unit=seconds displays 3600.000 (counting down in seconds).
  • Decimal Clipping: To round the decimal, pass clip=INTEGER. If the clip amount is larger than the decimal, the extra zeros are quietly trimmed (e.g., clip=4 on 12.5 becomes 12.5, not 12.5000).
  • Error Handling: If an unsupported unit is passed (e.g., unit=centuries), the timer will fail with a red error message.

10. Auto-Shrink Mode

If your timer displays a very large number (e.g., counting down in ms from 10 days), it might overflow the screen. You can opt-in to an auto-shrink feature.

  • Parameter: autoshrink=true
  • Behavior: When enabled, the font size will dynamically shrink as the number gets longer, ensuring it always fits within the screen boundaries. When disabled, the font stays at maximum size even if it gets clipped.
  • Default: autoshrink=false (Font stays large, and may overflow the screen).

11. Mouse Hide Mode

For a distraction-free, presentation-style timer, you can automatically hide the mouse cursor after a brief period of inactivity.

  • Parameter: mouse-hide=true
  • Behavior: The mouse cursor will fade away after 0.5 seconds of no movement. It will instantly reappear the moment you move your mouse again.
  • Default: mouse-hide=false (Mouse cursor is always visible).

12. Custom Font Upload

Upload your own font file to use for the timer display and label. The font will be applied instantly in the preview.

  • Supported formats: .woff2, .woff, .ttf, .otf
  • How it works: Upload any font file, and it will replace the timer display font
  • Note: The font is applied locally and will persist during your session
